# 建立專案 README

請分析目前的專案內容與原始碼結構，產生一份標準的 `README.md`，使用台灣用語正體中文撰寫。

## 內容結構

- 專案簡介（1 段）
- 功能重點（條列）
- 環境需求（含版本）
- 安裝步驟
- 使用方式（含最小可行範例）
- 開發流程（測試 / 格式化 / 建置）
- 常見問題（精簡）
- License

## 風格要求

- 使用客觀、事實性的語句描述功能與效果。
- 命令、路徑、檔名使用 code formatting。
- 條列簡潔，避免冗長段落。
- 若資訊不足，列在 `⚠️ 待補資訊` 區塊，不要臆測。

## License 規則

- 若專案已有 `LICENSE` / `LICENSE.md`，在 README 連結現有檔案。
- 若尚無授權檔，標註 `⚠️ 待確認：尚未設定授權`。
