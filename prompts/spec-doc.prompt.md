# 產生開發需求規格文件

讀取工作區 `.local/ai-sessions/clarify-output.md`，將其轉化為一份供人類同事閱讀與討論的開發需求規格文件。

- 若檔案不存在，提示使用者先以 `@Clarify` 完成需求釐清。

## 輸出格式

以下為固定章節順序，不得增減：

### 1. 需求概述

一到兩段說明本次需求的背景、目的與主要範圍。讓未參與討論的同事能快速掌握脈絡。

### 2. 功能需求

每項功能以以下格式呈現：

```
#### [功能名稱]

**說明**：描述此功能的目的與行為。

**驗收標準**：
- 具體可判斷的完成條件（行為層面描述）
```

### 3. 非功能需求（若有）

涵蓋效能、安全性、相容性、可維護性等面向的要求。若 clarify-output.md 未提及，略去此章節。

### 4. 排除範圍（若有）

明確列出本次不處理的項目，避免範圍蔓延的誤解。

## 風格要求

- 目標讀者為同事，使用自然語言撰寫，避免 AI 結構化清單的機械感。
- 每個功能需求必須有說明段落，**禁止只列標題或勾選清單（ - [ ] 格式）**。
- 驗收標準以行為描述為主，不包含實作細節或技術選型。
- 文件以中性客觀語氣撰寫（Context-Free），不涉及任務過程或時間軸。

## 產出

將文件存入工作區 `.local/ai-sessions/spec-doc.md`（目錄不存在時自動建立），並提示：

> 需求規格文件已儲存至 `.local/ai-sessions/spec-doc.md`。
